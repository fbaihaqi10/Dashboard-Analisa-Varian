Maintenance Guide

Update Data Periode Baru

Panduan menambahkan periode baru (mis. Juli 2026 dan seterusnya) ke dashboard ini — semua lewat browser github.com, tanpa install apa pun.

1Minta Claude membuatkan file JSON periode baru

Upload workbook .xlsb bulan tersebut (mis. "07. PL Seasonality ... Base Juli.xlsb") ke chat Claude, lalu minta: "tolong extract data periode Juli ini jadi format JSON yang sama seperti periode sebelumnya untuk dashboard saya". Claude akan menghasilkan satu file 2026-07.json dengan struktur yang sama persis seperti periode-periode sebelumnya.

Penting: pastikan workbook barunya masih pakai template yang sama (ada sheet Summary PL-All, Komparasi PL v Realisasi, Komposisi COGS, Variance PL per UB) — kalau strukturnya berubah, kabari Claude supaya disesuaikan dulu.

2Upload file JSON ke folder data/periods/

Di repo GitHub kamu, masuk ke folder data/periods → Add file → Upload files → drag file 2026-07.json yang baru dibuatkan Claude → Commit changes.

3Daftarkan periode baru di manifest.json

Buka data/periods/manifest.json → klik pensil (Edit) → tambahkan entri baru ke array periods, dan update default ke periode terbaru:

{
  "periods": [
    { "key": "2026-04", "label": "April", "monthIndex": 3 },
    { "key": "2026-05", "label": "Mei",   "monthIndex": 4 },
    { "key": "2026-06", "label": "Juni",  "monthIndex": 5 },
    { "key": "2026-07", "label": "Juli",  "monthIndex": 6 }
  ],
  "default": "2026-07"
}

monthIndex pakai basis 0 (Jan=0, Feb=1, … Jul=6, … Des=11) — dipakai untuk menandai titik bulan berjalan di halaman Seasonality. key harus sama persis dengan nama file JSON-nya (tanpa .json). Commit changes.

4Selesai — cek dashboard

Refresh link GitHub Pages kamu. Dropdown "Periode" di pojok kanan atas akan otomatis menampilkan periode baru begitu manifest.json ter-update. Tidak perlu ubah index.html sama sekali untuk menambah periode baru.

!Kalau ragu

Kirim saja workbook bulan barunya ke chat Claude dan minta dibuatkan file JSON + dipandu upload-nya persis seperti sebelumnya — tidak perlu menulis JSON manual.
